1.node

1.1. node是什么?

  • node中也包含js解析引擎(node现在用的就是V8引擎!，速度非常快)
  • node
  • php/后端使用的语言 (后端写的代码是服务器!)
    • 区分前端和后端(就看最终代码是在哪里执行的，在浏览器中执行，就是前端写的)

  • node 【其实是一个包含的js解析引擎(V8)的软件!】 （可以在后端使用）

    • 有了js解析引擎就会解析js代码

  • 浏览器的js

    • 可以通过js操作html
    • DOM(和操作html有关),BOM(和浏览器有关),ECMA(就是js语法规范)
  • node中的js
    • 没有DOM,没有BOM,有ECMA
    • window(也就是说没有wiondow对象)

1.3. node快速开始

  • 1.10.xxx,node就开源
    • 开源的话，别人就可以贡献源码！
    • 第三方社区: 觉得node更新太慢!
      • 第三方社区基于原来的node继续开发，在此基础上加了很新的东西
    • 合并: 4.x直接升到4.x、4.xxx、5.xx、6.xx、7.xx

1.4 node中引入模块，和暴露数据

  • 引入模块,使用node中提供的 require('./jakc.js')方法
  • 暴露数据使用的是 module.exports = {xxx}或者 exports.xxx = yyy

node中不仅包含js解析引擎，还包含服务器软件的功能

  • 我们电脑上只要安装了node,不需要其他的软件配置，就能够开启一个http服务

注意

file协议，默认是不能够发ajax请求的！

  • 服务器就是一台电脑(这个电脑有独立ip)
  • 我们的笔记本也是电脑(也可以把最终的项目代码放到笔记本上，但是一般不这样做)
  • 操作系统都可能是一样的
  • 服务器能运行的软件，我们电脑也能运行

环境变量!

  • 在cmd窗口中敲一个命令,本质上是执行了一个程序(软件)
  • 默认按下回车时，会到当前命令行所对应的目录寻找文件,如果找到立即执行；如果找不到呢?就会到环境变量中的PATH中指定的目录寻找!

例:我们把node的默认安装目录: C:\Program Files\nodejs,配置PATH上，那么
我们在cmd中敲命令时，如果当前目录找不到，就会到C:\Program Files\nodejs
目录寻找相应的文件执行!

命令行

  • 系统底层还是命令!
  • windows系统刚出来是没有界面的
  • 在cmd中敲命令，会默认到当前命令行对应的目录找文件执行

服务器就是一台电脑

  • 只是性能一般比我们普通电脑高!
